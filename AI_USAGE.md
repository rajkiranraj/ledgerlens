### Example 1: AI Generated Outdated React Imports

**Original AI Output**
```jsx
import React, { useState, useEffect } from 'react';

function Dashboard() {
  // ...
}
```

**Why It Was Wrong**
- React 19+ doesn't require explicit `import React from 'react'` for JSX
- The project uses ESLint which flags unused imports
- Adds unnecessary bytes to the bundle

**How It Was Detected**
- Ran `npm run lint` and immediately saw `react/no-unused-imports` errors
- Visually inspected the code and noticed the unused import

**Final Correction**
```jsx
import { useState, useEffect } from 'react';

function Dashboard() {
  // ...
}
```

### Example 2: AI Created Invalid Date Parsing Logic

**Original AI Output**
```python
from datetime import datetime

def parse_date(date_str):
    return datetime.strptime(date_str, '%Y-%m-%d').date()
```

**Why It Was Wrong**
- Only handles one date format (`YYYY-MM-DD`)
- Real-world CSV files use multiple formats (`DD/MM/YYYY`, `MM/DD/YYYY`, `DD-MMM-YYYY`)
- No error handling for unparseable dates

**How It Was Detected**
- Tested with sample CSV files containing different date formats
- The function threw `ValueError` exceptions for valid dates in other formats
- Manually reviewed the logic and realized it was too simplistic

**Final Correction**
```python
from datetime import datetime

def parse_date(date_str):
    date_formats = [
        '%Y-%m-%d',
        '%d-%m-%Y',
        '%d/%m/%Y',
        '%m/%d/%Y',
        '%d-%b-%Y'
    ]
    for fmt in date_formats:
        try:
            return datetime.strptime(date_str.strip(), fmt).date(), None
        except ValueError:
            continue
    return None, "Unparseable date format"
```

### Example 3: AI Suggested Insecure API Key Handling

**Original AI Output**
```javascript
// frontend/config.js
const API_KEY = 'nvapi-xxxxxxxxxxxxxxxxxxxxxxxxxx';
export default API_KEY;
```

**Why It Was Wrong**
- Exposes secret API key in client-side code
- API keys should **never** be in frontend repositories
- Violates security best practices

**How It Was Detected**
- Immediately recognized this as a critical security issue
- Security best practices are fundamental knowledge for any developer
- The backend already had proper AI integration, so frontend shouldn't need the key

**Final Correction**
- Removed all API key references from frontend
- Kept AI integration **100% server-side**
- Implemented a deterministic fallback in case AI is unavailable

### Example 4: AI Generated Buggy Debt Minimization Algorithm

**Original AI Output**
```python
def minimize_debts(balances):
    transactions = []
    debtors = sorted([(u, -b) for u, b in balances.items() if b < 0], key=lambda x: x[1])
    creditors = sorted([(u, b) for u, b in balances.items() if b > 0], key=lambda x: x[1])
    
    while debtors and creditors:
        debtor, debt = debtors[0]
        creditor, credit = creditors[0]
        
        amount = min(debt, credit)
        transactions.append((debtor, creditor, amount))
        
        debtors[0] = (debtor, debt - amount)
        creditors[0] = (creditor, credit - amount)
        
        if debtors[0][1] == 0:
            debtors.pop(0)
        if creditors[0][1] == 0:
            creditors.pop(0)
            
    return transactions
```

**Why It Was Wrong**
- Modifies list elements while iterating, leading to unexpected behavior
- Doesn't handle floating-point precision properly (critical for currency!)
- No handling for very small balances that should be considered zero

**How It Was Detected**
- Wrote unit tests with known inputs and expected outputs
- The algorithm produced incorrect results for complex balance scenarios
- Manually stepped through the logic and identified the issues

**Final Correction**
```python
def minimize_debts(balances):
    transactions = []
    active_balances = []
    
    for user, balance in balances.items():
        if abs(balance) >= 0.01:  # Ignore negligible balances
            active_balances.append({'user': user, 'balance': balance})
    
    debtors = sorted([x for x in active_balances if x['balance'] < 0], 
                     key=lambda x: x['balance'])
    creditors = sorted([x for x in active_balances if x['balance'] > 0], 
                       key=lambda x: -x['balance'])
    
    d_idx = 0
    c_idx = 0
    
    while d_idx < len(debtors) and c_idx < len(creditors):
        debtor = debtors[d_idx]
        creditor = creditors[c_idx]
        
        debt_amount = -debtor['balance']
        credit_amount = creditor['balance']
        
        settle_amount = min(debt_amount, credit_amount)
        settle_amount_rounded = round(settle_amount, 2)
        
        if settle_amount_rounded > 0:
            transactions.append({
                'from_user': debtor['user'],
                'to_user': creditor['user'],
                'amount': settle_amount_rounded
            })
        
        debtor['balance'] += settle_amount
        creditor['balance'] -= settle_amount
        
        if abs(debtor['balance']) < 0.01:
            d_idx += 1
        if abs(creditor['balance']) < 0.01:
            c_idx += 1
    
    return transactions
```

## Verification Process

All AI-generated code went through this strict verification process:

1. **ESLint Check**: Ran `npm run lint` on all frontend code
2. **Python Linting**: Ran `flake8` on all backend code
3. **Type Checking**: Verified TypeScript types (where applicable)
4. **Unit Tests**: Wrote tests for critical functions
5. **Manual Review**: Visually inspected every line of code
6. **Edge Case Testing**: Tested with malformed inputs, empty files, etc.
